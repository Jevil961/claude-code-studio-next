# Claude Code Studio Next

Claude Code Studio Next هو استوديو سطح مكتب متعدد المنصات مبني على Tauri لاستخدام Claude Code وإدارته. صمم للمستخدمين الذين يديرون مزودين متعددين، وهويات، ومهارات، وخدمات MCP، ومشاريع، وسجلات مهام طويلة، مع الحفاظ على استهلاك ذاكرة واضح وتنظيف جيد للعمليات.

## المشكلة التي يعالجها

Claude Code قوي، لكن الاستخدام اليومي قد يصبح مزعجا عندما تكون إعدادات المزودين والمهارات وخدمات MCP وسجل المشاريع والتشخيصات وعمليات المهام موزعة في أماكن مختلفة. يجمع هذا التطبيق هذه الجوانب في مساحة عمل مكتبية واحدة.

يعالج المشروع أيضا مشكلة أداء عملية: يجب ألا تبقى عمليات Claude Code و Node مفتوحة بعد انتهاء المهمة. يقوم المشغل بتشغيل Claude Code عند الحاجة فقط، ثم ينظف العمليات المرتبطة بعد انتهاء العمل.

## الميزات الأساسية

- إدارة المزودين وإعدادات النماذج المسبقة.
- تنظيم المهارات ومزامنتها حسب الهوية.
- إدارة خدمات MCP.
- تصفح مشاريع ومحادثات Claude Code.
- مشغل Claude Code مصمم لتقليل استهلاك الذاكرة.
- إحصاءات استخدام مع ذاكرة تخزين مؤقت.
- تصدير تشخيصات يتضمن المسارات والإصدارات والعمليات والأعداد والأخطاء الحديثة.
- نسخ احتياطية تلقائية قبل الكتابات الحساسة.
- حزم سطح مكتب عبر Tauri مع خلفية Node مخفية.

## دعم المنصات

يستهدف المشروع Windows و macOS و Linux. تم التحقق محليا من إصدار Windows x64. يتم بناء حزم macOS Intel و macOS Apple Silicon و Linux عبر GitHub Actions على بيئات أصلية. دعم ARM64 جزء من استراتيجية الإصدار، مع بناء أصلي لنظام macOS ARM64 ودعم على مستوى المصدر للأهداف الأخرى.

## التثبيت

ينبغي للمستخدمين تنزيل الحزم من GitHub Releases. على Windows يمكن استخدام المثبت أو الحزمة المحمولة. على macOS يستخدم ملف DMG. على Linux يمكن استخدام AppImage أو حزمة Debian عند توفرها.

يجب أن يكون Node.js 18 أو أحدث متاحا في `PATH`. يجب تثبيت Claude Code، وإذا لم يكن مثبتا فسيعرض التطبيق إرشادات الإعداد.

```

```

## الإصدار

لا ينبغي رفع الملفات الثنائية الناتجة إلى مستودع Git. يجب نشرها في GitHub Releases. يقوم workflow الخاص بالإصدار ببناء الحزم الأصلية عند دفع وسم يبدأ بـ `v*`.
