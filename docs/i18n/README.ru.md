# Claude Code Studio Next

Claude Code Studio Next — это кроссплатформенная настольная студия на Tauri для Claude Code. Она предназначена для пользователей, которым нужно управлять провайдерами, идентичностями, Skills, MCP-сервисами, проектами и историей задач, не теряя контроль над памятью и процессами.

## Какую Проблему Решает Проект

Claude Code мощный, но ежедневная работа становится шумной, когда настройки провайдеров, Skills, MCP-сервисы, история проектов, диагностика и процессы задач находятся в разных местах. Приложение собирает эти элементы в единое рабочее пространство.

Проект также решает практическую проблему производительности: процессы Claude Code и Node не должны оставаться активными после завершения задачи. Runner запускает Claude Code только на время работы и затем очищает связанные процессы.

## Основные Возможности

- Управление провайдерами и пресетами моделей.
- Организация и синхронизация Skills по идентичностям.
- Управление MCP-сервисами.
- Навигация по проектам и диалогам Claude Code.
- Runner Claude Code с учетом потребления памяти.
- Статистика использования с кэшем.
- Экспорт диагностики: пути, версии, процессы, счетчики и последние ошибки.
- Автоматические резервные копии перед опасными изменениями.
- Упаковка через Tauri со скрытым Node-бэкендом.

## Платформы

Проект ориентирован на Windows, macOS и Linux. Локально проверена сборка Windows x64. Пакеты для macOS Intel, macOS Apple Silicon и Linux собираются через GitHub Actions на нативных runner-средах. ARM64 входит в стратегию выпуска: macOS ARM64 собирается нативно, а другие ARM64-цели поддерживаются на уровне исходного кода и инфраструктуры сборки.

## Установка

Пользователям следует скачивать пакеты из GitHub Releases. Для Windows доступны установщик и portable zip. Для macOS используется DMG. Для Linux доступны AppImage или Debian-пакет, когда они опубликованы.

Node.js 18 или новее должен быть доступен в `PATH`. Claude Code должен быть установлен; если его нет, приложение покажет подсказки по настройке.

## Разработка

```powershell
npm install
npm run dev
```

Проверка:

```powershell
npm run check
npm test
cargo check --manifest-path src-tauri\Cargo.toml
```

## Выпуск

Сгенерированные бинарные файлы не нужно коммитить в Git. Они публикуются в GitHub Releases. Workflow выпуска создает нативные пакеты при отправке тега `v*`.

## Текущий Статус

Версия `0.1.0` готова к публикации на GitHub. Node.js остается документированной runtime-зависимостью. Подпись кода и автообновления пока не включены.
